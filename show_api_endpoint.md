1. 口座管理 (Account Management)
====================================================================================================
説明: 口座情報の取得と管理

----------------------------------------------------------------------------------------------------

  [1-1] 口座購読情報の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /account/subscriptions
  SDKメソッド    : api.account.get_app_subscriptions()
  説明           : APIアプリケーションに紐づく口座情報の一覧を取得
  パラメータ     : なし
  レスポンス     : 口座ID、購読ID、ステータスなど


  [1-2] 口座詳細情報の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /account/{account_id}
  SDKメソッド    : api.account.get_account_detail(account_id)
  説明           : 指定した口座の詳細情報を取得
  パラメータ     :
                   - account_id: 口座ID
  レスポンス     : 口座詳細情報


  [1-3] 口座残高の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /account/{account_id}/balance
  SDKメソッド    : api.account.get_account_balance(account_id)
  説明           : 口座の現金残高、購買力、総資産額などを取得
  パラメータ     :
                   - account_id: 口座ID
  レスポンス     : 現金残高、購買力、総資産額など


  [1-4] 口座ポジションの取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /account/{account_id}/positions
  SDKメソッド    : api.account.get_account_positions(account_id)
  説明           : 保有している株式ポジション情報を取得
  パラメータ     :
                   - account_id: 口座ID
  レスポンス     : 銘柄、数量、平均取得価格、現在価格など

2. 注文管理 (Order Management)
====================================================================================================
説明: 注文の作成、変更、キャンセル、照会

----------------------------------------------------------------------------------------------------

  [2-1] 注文の作成
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : POST
  パス           : /order
  SDKメソッド    : api.order.place_order(account_id, order_params)
  説明           : 新規注文を作成(成行、指値、逆指値など)
  パラメータ     :
                   - account_id: 口座ID
                   - order_params: 注文パラメータ(銘柄、数量、価格、注文タイプなど)
  レスポンス     : 注文ID、クライアント注文ID、ステータス


  [2-2] 注文の変更
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : PUT
  パス           : /order/{order_id}
  SDKメソッド    : api.order.modify_order(account_id, order_id, new_params)
  説明           : 既存注文の価格や数量を変更
  パラメータ     :
                   - account_id: 口座ID
                   - order_id: 注文ID
                   - new_params: 変更パラメータ
  レスポンス     : 変更後の注文情報


  [2-3] 注文のキャンセル
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : DELETE
  パス           : /order/{order_id}
  SDKメソッド    : api.order.cancel_order(account_id, order_id)
  説明           : 指定した注文をキャンセル
  パラメータ     :
                   - account_id: 口座ID
                   - order_id: 注文ID
  レスポンス     : キャンセル結果


  [2-4] 注文詳細の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /order/{order_id}
  SDKメソッド    : api.order.query_order_detail(account_id, client_order_id)
  説明           : 指定した注文の詳細情報を取得
  パラメータ     :
                   - account_id: 口座ID
                   - client_order_id: クライアント注文ID
  レスポンス     : 注文の詳細情報、ステータス、約定情報など


  [2-5] 注文一覧の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /orders
  SDKメソッド    : api.order.query_orders(account_id, params)
  説明           : 口座の注文一覧を取得(フィルタ可能)
  パラメータ     :
                   - account_id: 口座ID
                   - params: フィルタパラメータ(日付、ステータスなど)
  レスポンス     : 注文のリスト


  [2-6] 未約定注文の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GET
  パス           : /orders/open
  SDKメソッド    : api.order.get_open_orders(account_id)
  説明           : 現在有効な未約定注文を取得
  パラメータ     :
                   - account_id: 口座ID
  レスポンス     : 未約定注文のリスト

3. マーケットデータ (Market Data) - GRPC
====================================================================================================
説明: 株式情報とマーケットデータの取得
注意: ※現在、HTTP経由のマーケットデータリクエストは未サポート。GRPCプロトコルを使用。

----------------------------------------------------------------------------------------------------

  [3-1] 銘柄情報の取得
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GRPC
  パス           : /instrument
  SDKメソッド    : grpc_api.instrument.get_instrument(symbols, category)
  説明           : 銘柄コードリストから銘柄の基本情報を取得
  パラメータ     :
                   - symbols: 銘柄コードのリスト (例: ['AAPL', 'TSLA'])
                   - category: カテゴリ (例: 'US_STOCK')
  レスポンス     : 銘柄名、ISIN、取引所、セクターなど
  頻度制限       : 60回/分


  [3-2] マーケットスナップショット
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GRPC
  パス           : /market-data/snapshot
  SDKメソッド    : grpc_api.market_data.get_snapshot(symbols, category)
  説明           : 銘柄の最新価格情報をバッチ取得
  パラメータ     :
                   - symbols: 銘柄コードのリスト
                   - category: カテゴリ
  レスポンス     : 最新価格、出来高、高値、安値など
  頻度制限       : 1回/秒


  [3-3] ローソク足データ
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GRPC
  パス           : /market-data/bars
  SDKメソッド    : grpc_api.market_data.get_bars(symbol, category, timeframe, count)
  説明           : 指定期間のローソク足データを取得
  パラメータ     :
                   - symbol: 銘柄コード
                   - category: カテゴリ
                   - timeframe: 時間足(1m, 5m, 1d等)
                   - count: データ件数
  レスポンス     : OHLCV(始値、高値、安値、終値、出来高)データ

4. リアルタイム購読 (Real-time Subscriptions)
====================================================================================================
説明: 注文ステータスとマーケットデータのリアルタイム受信

----------------------------------------------------------------------------------------------------

  [4-1] 注文イベント購読
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : GRPC
  パス           : /trade-events
  SDKメソッド    : EventsClient.do_subscribe([account_ids])
  説明           : 注文ステータス変更のリアルタイム通知を受信
  パラメータ     :
                   - account_ids: 監視する口座IDのリスト
  レスポンス     : 注文作成、約定、キャンセル等のイベント通知


  [4-2] マーケットデータ購読
  ────────────────────────────────────────────────────────────────────────────────────────────────
  HTTPメソッド    : MQTT
  パス           : /quotes/subscribe
  SDKメソッド    : DefaultQuotesClient.connect_and_loop_forever()
  説明           : 銘柄の価格変動をリアルタイムで受信
  パラメータ     :
                   - symbol: 監視する銘柄コード
                   - category: カテゴリ
                   - subscribe_type: 購読タイプ(SNAPSHOT等)
  レスポンス     : リアルタイムの価格更新情報

重要な注意事項
====================================================================================================

1. 米国株の全銘柄リストを直接取得するAPIは存在しません
   → 銘柄コードを指定して個別に情報を取得する必要があります

2. 入出金履歴の取得APIは未実装です
   → Webullモバイルアプリまたはカスタマーサポートへの問い合わせが必要

3. マーケットデータのHTTPリクエストは現在未サポート
   → GRPCプロトコルを使用する必要があります

4. APIキーの有効期限はデフォルトで45日間
   → 期限切れ前にリセットが必要です

5. API呼び出しには頻度制限があります
   → 各エンドポイントの制限を確認してください
